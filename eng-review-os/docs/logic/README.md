# Logic

Put here:

- **Domain model** — entities, relationships, invariants (see `domain-model.md`).
- **Flows** — e.g. “create cycle → set goals → add evidence → export”; add `flows/` if you split diagrams.
- **Decisions** — short ADRs (`decisions/` optional) when a technical or product choice matters later.

Requirements belong in `../requirements/`; this folder is for **how the domain works**, not the commercial “why.”
